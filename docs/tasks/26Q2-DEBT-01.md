#### Context

> All 11 test files are flat in `tests/`. The source code is organized into `tools/` and `utils/` subdirectories, but tests don't mirror this structure.

**Current State**:
- `tests/` contains 11 test files with no subdirectory organization
- `tests/fixtures/` exists with test fixture files
- 3 test files reference fixtures via `Path(__file__).parent / "fixtures"`

**Investigation**:
```bash
# Current test files
find tests -name "test_*.py" | wc -l
# Output: 11

# Files referencing fixtures directory
grep -rl "fixtures" tests/test_*.py
# Output: test_compile.py, test_pdf_info.py, test_server_integration.py
```

#### Implementation Notes

- Fixture paths use `Path(__file__).parent / "fixtures"` — moving files one level deeper requires changing to `Path(__file__).parent.parent / "fixtures"`
- pytest discovers tests recursively under `testpaths = ["tests"]`, so no config changes needed
- No cross-imports between test files, so moves are independent
