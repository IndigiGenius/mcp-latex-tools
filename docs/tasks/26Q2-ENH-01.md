#### Acceptance Criteria

**compile.py — New Parameters**:
- [ ] `compile_latex()` accepts `engine: str` param with default `"pdflatex"`
- [ ] `compile_latex()` accepts `passes: int | str` param with default `1`
- [ ] `SUPPORTED_ENGINES` constant contains `("pdflatex", "xelatex", "lualatex", "latexmk")`
- [ ] Invalid `engine` value raises `CompilationError` mentioning "engine"
- [ ] Invalid `passes` value (e.g., `0`) raises `CompilationError` mentioning "passes"

**compile.py — Engine Support**:
- [ ] `engine="pdflatex"` uses `pdflatex` binary (default, backwards compatible)
- [ ] `engine="xelatex"` uses `xelatex` binary and produces PDF
- [ ] `engine="lualatex"` uses `lualatex` binary and produces PDF
- [ ] `engine="latexmk"` uses `latexmk -pdf` and produces PDF

**compile.py — Multi-Pass**:
- [ ] `passes=1` runs a single compilation pass (default)
- [ ] `passes=2` runs the engine twice
- [ ] `passes=3` runs the engine three times
- [ ] `passes="auto"` runs once, checks log for rerun suggestion, reruns if needed (max 3)
- [ ] `passes="auto"` with `simple.tex` (no cross-refs) runs exactly 1 pass
- [ ] `passes="auto"` with cross-references runs at least 2 passes
- [ ] `engine="latexmk"` handles passes automatically regardless of `passes` param

**compile.py — Bibliography**:
- [ ] Detects `\bibliography{}` in .tex → runs `bibtex` between passes
- [ ] Detects `\addbibresource{}` in .tex → runs `biber` between passes
- [ ] No bibliography in .tex → no `bibtex`/`biber` subprocess called

**CompilationResult — New Fields**:
- [ ] `CompilationResult` has `engine: Optional[str]` field (default `None`)
- [ ] `CompilationResult` has `passes_run: Optional[int]` field (default `None`)

**server.py — Schema Update**:

- [ ] `_handle_compile` passes `engine` and `passes` to `compile_latex()`

#### Implementation Notes

- `pdflatex`, `xelatex`, `lualatex` share CLI interface (`-interaction=nonstopmode -output-directory`)
- `latexmk` uses `-pdf -interaction=nonstopmode -output-directory=` (note `=` sign)
- Use `log_parser.parse_latex_log()` for auto-pass rerun detection (already exists)
- Bibliography detection: parse .tex for `\bibliography{}` → bibtex, `\addbibresource{}` → biber
- `bibtex` runs as `bibtex <stem>` in output dir; `biber` runs as `biber <output_dir>/<stem>`
- `_handle_compile` currently passes args positionally — switch to lambda/keyword args for new params
- Existing tests must continue to pass unchanged (backwards compatibility)
